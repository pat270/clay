a3-inputs

Inputs

This recipe will modify the input class, .form-control, in Clay CSS to match Material Design’s Filled Text Field shown in https://material.io/components/text-fields/. The Material Design site uses several HTML elements with JavaScript to create the style. We will try to duplicate it using only the input element.

We won’t be able to duplicate floating input labels without access to the markup so it won’t be covered here.

Input Default

The simplified Material Design styles for the default state are listed below:

.mdc-text-field__input {
    border-bottom: 1px solid;
    font-size: 1rem;
    font-weight: 400;
    letter-spacing: .009375em;
    padding: 20px 16px 6px;
    transition: opacity 150ms cubic-bezier(0.4, 0, 0.2, 1);
}

.mdc-text-field:not(.mdc-text-field--disabled) .mdc-text-field__input {
    color: rgba(0,0,0,.87);
}

.mdc-text-field:not(.mdc-text-field--disabled) .mdc-text-field__input {
    border-bottom-color: rgba(0,0,0,.42);
}

.mdc-text-field:not(.mdc-text-field--disabled) {
    background-color: #f5f5f5;
}

.mdc-text-field {
    border-radius: 4px 4px 0 0;
    height: 56px;
}

.mdc-line-ripple {
    height: 2px;
    transition: transform 180ms cubic-bezier(0.4, 0, 0.2, 1),opacity 180ms cubic-bezier(0.4, 0, 0.2, 1);
}

.mdc-text-field .mdc-line-ripple {
    background-color: #6200ee;
}

.mdc-text-field::before {
    transition: opacity 15ms linear,background-color 15ms linear;
    z-index: 1;
}

We have the styles we need to update the default state of form-control in Clay CSS. HTML inputs are generally vertically centered by default with the exception of textarea and select[multiple]. Setting the height should cover most use cases and $input-padding-y should not need to be updated.

In this case, Material Design sets a large value for padding-top which we will skip for now. Setting $input-padding-y to 20px will cut off text on our select input we will update case by case as necessary.

_/src/css/_clay_variables.scss_

$input-height: 3.5rem;

$input-bg: $gray-100;
$input-border-bottom-width: 0.0625rem;
$input-border-left-width: 0;
$input-border-right-width: 0;
$input-border-top-width: 0;
$input-border-color: rgba($black, 0.42);
$input-border-radius: 0.25rem 0.25rem 0 0;
$input-box-shadow: none;
$input-color: rgba($black, 0.87);
$input-padding-x: 1rem;

Clay CSS doesn’t have a variable to modify letter-spacing in form-control. We will need to update _clay_custom.scss. We can go a step further and add a $input-letter-spacing variable.

_/src/css/_clay_variables.scss_

$input-letter-spacing: 0.009375em;

_/src/css/_clay_custom.scss_

.form-control {
    letter-spacing: $input-letter-spacing;
}

Material Design applies several transitions when state changes on the input. We can set them using the $input-transition variable. We will use the background properties to apply the ripple effect later.

$input-transition: background-color 15ms linear, background-position 180ms cubic-bezier(0.4, 0, 0.2, 1), background-size 180ms cubic-bezier(0.4, 0, 0.2, 1), border-color 180ms cubic-bezier(0.4, 0, 0.2, 1);

Input Hover

Material Design’s input hover styles are below:

.mdc-text-field:hover::before {
    opacity: .04;
}

.mdc-text-field::before {
    background-color: rgba(0,0,0,.87);
}

.mdc-text-field:not(.mdc-text-field--disabled) .mdc-text-field__input:hover {
    border-bottom-color: rgba(0,0,0,.87);
}

Clay CSS doesn’t provide any variables for setting the hover state. We can add a couple variables so we can change them later and also add our overwrite.

_/src/css/_clay_variables.scss_

$input-hover-bg: $gray-200;
$input-hover-border-color: rgba($black, 0.87);

_/src/css/_clay_custom.scss_

.form-control {
    &:hover {
        background-color: $input-hover-bg;
        border-color: $input-hover-border-color;
    }
}

Input Focus

Material Design’s input focus styles are below:

.mdc-text-field.mdc-ripple-upgraded--background-focused::before, .mdc-text-field:not(.mdc-ripple-upgraded):focus::before {
    opacity: .12;
    transition-duration: 75ms;
}

.mdc-text-field::before, .mdc-text-field::after {
    background-color: rgba(0,0,0,.87);
}

Since we added the selector .form-control:hover in the previous example, we will need to redeclare background-color and border-color focus styles to overcome specificity. We will also add our ripple effect using background properties.

_/src/css/_clay_variables.scss_

$input-focus-bg: $gray-300;
$input-focus-border-color: transparent;
$input-focus-box-shadow: none;

_/src/css/_clay_custom.scss_

.form-control {
    background-position: 53% calc(100% + #{$input-border-bottom-width});
    background-repeat: no-repeat;
    background-size: 50% 2px;

    &:focus {
        background-color: $input-focus-bg;
        background-image: linear-gradient($primary-a700 100%, transparent 0%);
        background-size: 100% 2px;
        border-color: $input-focus-border-color;
    }
}
